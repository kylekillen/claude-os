# First Principles Thinking

A three-phase framework for breaking through conventional thinking and reasoning from fundamentals.

## When to Use

- Facing a problem where conventional solutions feel inadequate
- Questioning whether assumed constraints are actually real
- Need to challenge inherited beliefs or industry norms
- Want to find radical rather than incremental improvements

## The Three Phases

### Phase 1: DECONSTRUCT

Break the problem into its fundamental components.

Ask:
- What is this really made of?
- What are the actual raw materials, costs, and elements?
- What would this look like if built from scratch today?

Output a list of the true constituent parts, stripped of assumed packaging.

### Phase 2: CHALLENGE

Classify every constraint into one of three categories:

| Type | Definition | Example |
|------|------------|---------|
| **Hard** | Physics, math, immutable laws | Gravity, speed of light, thermodynamics |
| **Soft** | Policy choices, regulations, conventions | "We've always done it this way", industry standards |
| **Assumptions** | Unvalidated beliefs | "That would be too expensive", "Users won't accept that" |

For each constraint, ask: "Is this actually immovable, or just inherited?"

### Phase 3: RECONSTRUCT

Build a new solution using ONLY validated hard constraints.

Rules:
- Ignore form, optimize function
- Don't carry forward soft constraints unless deliberately chosen
- Start from "what would we build if nothing existed?"

## Example: SpaceX Rocket Cost

**Deconstruct:** Rockets are made of aluminum, titanium, copper, carbon fiber. Raw material cost: ~2% of rocket price.

**Challenge:** "Rockets are expensive" → ASSUMPTION (98% of cost is manufacturing choices, not materials). "Rockets are single-use" → SOFT constraint (engineering choice, not physics).

**Reconstruct:** Build reusable rockets, vertical landing, in-house manufacturing. Result: 10x cost reduction.

## How to Apply

When Kyle presents a problem:

1. State the problem clearly
2. Run DECONSTRUCT: List all components
3. Run CHALLENGE: Classify each constraint (Hard/Soft/Assumption)
4. Run RECONSTRUCT: Propose solution from fundamentals only
5. Present findings with explicit constraint classifications

Ask Kyle which phase to start with, or run all three in sequence.
