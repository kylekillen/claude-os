# The Algorithm

A 7-phase scientific method for executing complex tasks systematically.

## When to Use

- Complex tasks with multiple steps or unknowns
- When you need to ensure nothing is missed
- Tasks requiring verification before declaring success
- When Kyle wants rigorous execution, not quick approximation

## The 7 Phases

Execute IN ORDER. Each phase builds on the previous.

### Phase 1: OBSERVE

Understand the request and establish foundation.

- What is Kyle actually asking for?
- What context exists (files, constraints, history)?
- What are the explicit and implicit requirements?

**Output:** List of requirements and constraints discovered.

### Phase 2: THINK

Ensure comprehensive coverage.

- What did OBSERVE miss?
- Are there hidden requirements?
- What could go wrong?
- What assumptions am I making?

**Output:** Refined requirements list with gaps filled.

### Phase 3: PLAN

Sequence the work.

- What order should things happen?
- What tools/capabilities are needed for each step?
- What are the dependencies between steps?

**Output:** Ordered task list with approach for each.

### Phase 4: BUILD

Make criteria testable.

- How will I know each step succeeded?
- What does "done" look like for each requirement?
- Convert abstract goals to specific, verifiable conditions.

**Output:** Success criteria for each task.

### Phase 5: EXECUTE

Do the actual work.

- Work through the plan step by step
- Track progress against success criteria
- Adapt if obstacles arise (but document changes)

**Output:** Completed work with status for each task.

### Phase 6: VERIFY

Test skeptically.

- Does each deliverable actually meet its success criteria?
- Did I miss anything?
- Would this pass review by someone skeptical?

**Output:** Verification results, issues found.

### Phase 7: LEARN

Deliver and capture insights.

- Present organized results to Kyle
- Note what worked well
- Note what to do differently next time

**Output:** Final deliverable + lessons learned.

## How to Apply

When Kyle invokes the algorithm:

1. Announce you're using the 7-phase approach
2. Run each phase explicitly, showing output
3. Don't skip phases even if they seem obvious
4. If a phase reveals problems, you may loop back (but note it)
5. Complete with LEARN phase summary
